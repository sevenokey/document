在 Codex 里，Skills（技能） 一般指的是一种 可复用、可配置的能力模块，用来让 Codex 在特定场景下“会做某一类事”。不同平台/版本对 Skills 的具体实现略有差异，但核心作用是类似的。

下面用通用、产品层面的解释帮你理解 👇

一句话理解

Skills = 给 Codex 加“专业能力插件”
让它在某些任务上表现得更稳定、更可控，而不是每次都从零推理。

Skills 通常能做什么？
1️⃣ 约束 Codex 的行为方式

Skills 会定义：

能做什么（能力范围）

不能做什么（边界）

用什么格式输出结果

遵循什么规则或流程

比如：

只能写某种语言的代码

必须按公司规范生成 API

回答时只引用内部文档

2️⃣ 提供“专用能力”

一个 Skill 通常专注于 单一任务或一类任务，例如：

写前端组件

生成 SQL 查询

代码 Review

单元测试生成

文档摘要

工单分类

相比直接对话，用 Skill 更稳定、结果更一致。

3️⃣ 复用 & 组合

Skills 的价值之一是：

一次定义，多处使用

可以被不同的 Agent / Flow / Prompt 调用

可以组合多个 Skills 完成复杂任务

例如：

「代码生成 Skill」+「安全检查 Skill」+「格式化 Skill」

和普通 Prompt 的区别？
对比项	Prompt	Skill
临时性	一次性	可长期复用
稳定性	依赖输入质量	行为更可控
适合谁	个人使用	团队 / 产品级
是否结构化	通常是自然语言	通常有配置或模板

👉 可以理解为：
Prompt 更像聊天，Skill 更像“功能”

常见使用场景

内部开发工具（Dev Copilot）

自动化流程（CI / 工单 / 运维）

企业级 AI 应用

多 Agent 系统中的能力模块
